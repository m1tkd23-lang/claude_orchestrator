# docs/将来フロー_標準ラインと改善ライン統合仕様.md

## 1. 目的

標準開発フローと改善提案フローを統合し、
自走可能な開発サイクルを構築する。

---

## 2. 全体フロー

```text
task
  ↓
task_router
  ↓
implementer
  ↓
reviewer
  ↓
director
  ├─ revise → task_router
  ├─ stop   → 終了
  └─ approve
        ↓
   planner_safe
        ↓
   planner_improvement
        ↓
   plan_director
        ├─ no_adopt → 終了
        └─ adopt
             ↓
        task自動生成
             ↓
        task_router
             ↓
        次サイクルへ
3. フローの役割分離
3-1. 標準ライン

task_router

implementer

reviewer

director

役割：
→ 与えられた task を完了させる

3-2. 改善ライン

planner_safe

planner_improvement

plan_director

役割：
→ 次にやるべき task を決める

4. planner の役割
planner_safe

既存タスクの延長線

低リスク

高優先度

planner_improvement

改善提案

中〜高リスク

将来的価値

5. plan_director の役割

proposal の評価

1件採用 or 不採用

自動 task 化の最終判断

6. 自動化レベル
レベル1（現在）

planner は手動実行

proposal は人が選択

レベル2（次段階）

director approve 後に planner 自動起動

plan_director は手動実行

レベル3（将来）

plan_director 自動実行

safe proposal のみ自動採用

improvement は人確認

7. 安全制御

1サイクル1タスクのみ生成

threshold 未満は実行しない

depends_on 未解決は除外

8. メリット

開発が止まらない

改善が自然に回る

人の判断ポイントを削減可能

9. リスクと対策
リスク	対策
無限ループ	max_cycles 制御
不適切タスク生成	threshold
暴走	safe のみ自動化
10. 今後の実装ステップ

workflow に planner フェーズ追加
