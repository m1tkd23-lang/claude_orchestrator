## 実行ルール
- used_skills には実際に使った skill 名だけを配列で入れてください
- assigned skills に書かれていても、使っていない skill は used_skills に入れないでください
- assigned skills が空なら used_skills は空配列でもかまいません
- summary は空にせず、最終判断の理由を具体的に書いてください
- final_action が revise の場合は、next_actions を空配列にせず、次サイクルで何を直すか具体的に書いてください
- final_action が approve の場合は、完了判断の根拠が summary から読めるようにしてください
- final_action が stop の場合は、停止理由が summary から読めるようにしてください
- director は自分で実装せず、判断に徹してください

## 重要な実行指示
- 判断結果の JSON を作成するだけでなく、必ず指定された保存先にファイルとして保存してください
- 保存先は必ずこの task の対象 repo 配下の .claude_orchestrator を使用してください
- 親repoや別repoの .claude_orchestrator に保存してはいけません
- ファイル保存が成功してから応答を返してください

## output rules
- JSON のみを返してください
- コードフェンスは禁止です
- 前置き文は禁止です
- role は "director" にしてください
- task_id と cycle は入力と一致させてください
