# 開発の目的本筋

## 目的

本システムは、AI（Claude）を用いてタスクを段階的に処理し、
安全かつ再現性のある開発フローを実現することを目的とする。

---

## 中核フロー

TASKは以下の順で処理される。

task_router → implementer → reviewer → director

これを1サイクルとし、必要に応じて複数サイクル繰り返す。

---

## 重要ルール

- 全TASKは必ず task_router から開始する
- reviewer の needs_fix は implementer に直接戻らない
- director が revise を出した場合のみ次サイクルへ進む
- state.json が進行状態の正本である
- task.json は task_router により補強される

---

## planner の位置づけ

planner は本流ではなく、

「completed task から次の task 候補を提案する外部機構」

である。

---

## 各データの役割

### task.json
- タスク定義
- task_router により補強される

### state.json
- 進行状態の管理
- role遷移を決定する

### report（各role）
- 各roleの成果物
- 次roleへの入力となる

### planner proposal
- 次task候補
- 人またはGUIが選択してtask化する

---

## 壊してはいけないもの

- workflow（role遷移）
- state管理
- schema整合性
- report形式
- ファイル保存先ルール（.claude_orchestrator配下）

---

## carry_over フロー

前 task の未解決リスクを次 task に引き継ぐ仕組みを carry_over と呼ぶ。

### 転記元

前 task の `director_report_v{N}.json` の `remaining_risks` フィールド。

### 転記先

次 task の **`task.json`** の `initial_execution_notes` フィールドの **先頭**。

### プレフィックス形式

```
[carry_over from TASK-XXXX] {remaining_risks の内容}
```

`XXXX` には転記元の task_id の番号部分を入れる。元の `remaining_risks` の文言は改変しない。

### 詳細仕様

詳細手順・制約・v1 拡張方針は `.claude_orchestrator/docs/operations/carry_over_guide.md` を参照すること。

---

## 非目的

- AIが全てを自律的に決定すること
- 無制限な探索や拡張
- workflowを無視した直接実装
